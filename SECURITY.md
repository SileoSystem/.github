# Security Policy

## Supported Versions

Only the following versions of Selio are supported with security updates:

| Version        | Supported |
|----------------|-----------|
| Latest release | ✅         |
| `main` branch  | ✅         |
| Older releases | ❌         |

Security fixes are not guaranteed for unsupported versions.

---

## Reporting a Vulnerability

All security issues **must be reported through GitHub**.

Please use the **Security Advisory / Vulnerability reporting feature** of the affected repository.  
Do **not** open regular issues for security-related problems.

When reporting a vulnerability, include:

- A clear and concise description of the issue
- Affected repository and component
- Steps to reproduce, if applicable
- Potential impact and severity
- Any relevant logs, screenshots, or proof-of-concept code

Reports that lack sufficient technical detail may be closed or require follow-up.

---

## What to Expect

- Reports are reviewed by maintainers
- Valid vulnerabilities will be triaged and prioritized
- Fixes are developed and reviewed before disclosure
- Updates are provided through GitHub once available

Response times depend on severity and complexity.

---

## Disclosure Guidelines

- Do not publicly disclose vulnerabilities before a fix is released
- Coordinate disclosure through GitHub maintainers
- Responsible disclosure is expected

---

## Scope

### In Scope
- Selio-owned repositories
- Security, privacy, and integrity issues
- Authentication, authorization, and cryptography flaws
- Data leaks, privilege escalation, or sandbox escapes

### Out of Scope
- Vulnerabilities in third-party dependencies unless directly exploitable
- Denial-of-service via unrealistic traffic
- Social engineering attacks
- Issues requiring physical device access

---

## Legal

All contributions and fixes are handled under the **AGPL-3.0** license.

By reporting a security issue, you confirm that:
- The report is made in good faith
- No laws or user privacy were violated during testing

---

Security is a shared responsibility.  
Clear reports help protect users who value privacy and independence.
